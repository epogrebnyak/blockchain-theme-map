# Blockchain and Digital Assets Theme Map


```yaml
# Blockchain and Digital Assets Theme Map

# ============================================
# MINIMAL SET (6 critical categories)
# ============================================

# 1. User-facing use cases
Use cases:
  Financial:
    - Payments
    - Trading and derivatives
    - Existing RWA
    - New securities

  Physical:
    - Wireless networks (Helium)
    - Storage (Filecoin, Arweave)
    - Compute (Render, Akash)
    - Supply chain and provenance

  Social and governance:
    - Voting rights and DAOs
    - Identity
    - Reputation and credentials

  Entertainment:
    - Gaming
    - Collectibles
    - Betting and prediction markets

# 2. On-chain activity flow
Transactions:
  Create (new):
    - Native issuance (block rewards, staking and validation)
    - Token minting
    - Airdrops and rewards

  Acquire (entry):
    - At the table and P2P
    - Fiat on-ramp (broker, ATM and card)

  Exchange (secondary):
    - Centralised exchanges (CEX)
    - Decentralised protocols (DEX)
    - Aggregators and solvers
    - Cross-chain bridges

# 3. Asset types
Assets:
  - Native currency
  - Fungible
  - Non-fungible (NFTs)
  - Semi-fungible
  - Soulbound (non-transferable)
  - Off-chain data (oracles and storage references)

# 4. Economic models
Incentives:
  - Fee models (gas and priority fees)
  - MEV (order flow, arbitrage and liquidations)
  - Staking yields and rewards
  - Tokenomics (inflation and burn)

# 5. Networks
Networks:
  - Bitcoin (BTC)
  - Ethereum (ETH)
  - L2 and L3 Rollups
  - Alternative L1s (BNB, Solana, Tron, Sui and Aptos)
  - Private and consortium chains

# 6. Core technology
Foundations:
  - Cryptography (hashing, digital signatures and Merkle trees)
  - Address space and block data structures
  - Peer-to-peer networking (node discovery and gossip protocols)
  - Consensus mechanisms (participants, rules and guarantees)
  - Execution environments (VMs and state transitions)
  - API and user interface


# ============================================
# NETWORK ENHANCEMENTS (3 categories)
# ============================================

# 7. Cross-network communication
Interoperability:
  - Bridges
  - Cross-chain messaging
  - Light clients

# 8. External data
Oracles and data feeds:
  - Price feeds (Chainlink and Pyth)
  - Randomness (VRF)
  - Cross-chain data
  - Real-world event data (sports, weather and elections)
  - Data aggregation and verification

# 9. Data persistence
Storage:
  - On-chain storage (state and balances)
  - Off-chain storage (IPFS, Arweave and Filecoin)
  - Data availability layers (Celestia and EigenDA)
  - Decentralized databases (Ceramic and OrbitDB)


# ============================================
# FEATURES AND PROTECTION (3 categories)
# ============================================

# 10. User protection
Privacy:
  - Zero-knowledge proofs (ZK-SNARKs and ZK-STARKs)
  - Stealth addresses
  - Confidential transactions
  - Private voting
  - Encrypted metadata

# 11. System protection
Security:
  - Smart contract audits
  - Multi-sig and custody
  - Bug bounties
  - Formal verification
  - MEV protection

# 12. Compliance
Compliance:
  - KYC and AML
  - Sanctions screening
  - Transfer restrictions
  - Travel rule
  - Reporting and tax
```
